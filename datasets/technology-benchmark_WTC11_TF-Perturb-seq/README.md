Cross-technology comparison of 5 WTC11 TF-Perturb-seq datasets processed through the IGVF CRISPR Pipeline.

## Source Modules

Reusable analysis code lives in `src/tf_perturb_seq/`. The benchmark `bin/` scripts call these modules.

| Module | Location | Purpose |
|---|---|---|
| **QC** | `src/tf_perturb_seq/qc/` | Per-dataset QC from `inference_mudata.h5mu` (gene mapping, guide capture, intended target knockdown) |

### Step 1: Get data (`bin/1_get_data/`)

Downloads pipeline outputs from GCP and generates manifest files that drive all downstream steps.

**Entry point:** `bin/1_get_data/1_sync_benchmark.sh` — a runbook of commented-out commands. Uncomment and run the block for your desired pipeline run.

**Core script:** `bin/1_get_data/scripts/sync_benchmark.sh` — the sync orchestrator.

```bash
# Usage
bash scripts/sync_benchmark.sh \
    --gcp-prefix gs://igvf-pertub-seq-pipeline-data/scratch/bioinfolucas/<run> \
    --run-label <run-label> \
    [--dry-run] [--skip-download] [--force-download]
```

**What it does (3 phases):**

1. **Check GCP availability** — For each of the 5 datasets, checks whether the GCP run has `pipeline_dashboard/` (preferred, includes pre-computed QC) or only `pipeline_outputs/` (fallback, h5mu only). Missing datasets are skipped.

2. **Download** — Calls `scripts/sync_gcp_run.sh` per dataset. Dashboard datasets get `pipeline_dashboard/` synced; fallback datasets get `pipeline_outputs/inference_mudata.h5mu`.

3. **Generate manifests** — Writes two TSVs to `manifests/`:
   - `<run-label>_qc_input.tsv` — Lists fallback datasets that need local QC (input h5mu, output dir, run name). Empty if all datasets had dashboards. Used by `bin/2_qc/scripts/qc_array.sh`.
   - `<run-label>_qc_paths.tsv` — Maps every available dataset to its QC output files (gene metrics, guide metrics, intended target, trans). Dashboard datasets point to `pipeline_dashboard/additional_qc/`; fallback datasets point to `pipeline_qc/` (populated after running local QC).

**GCP-to-local dataset name mapping** (hardcoded in `sync_benchmark.sh`):

|---|---|
| GaryHonDataset | Hon_WTC11-benchmark_TF-Perturb-seq |
| HuangFuDataset | Huangfu_WTC11-benchmark_TF-Perturb-seq |

**Local directory structure after sync:**

### Step 2: QC comparison (`bin/2_qc/`)

Cross-technology QC comparison notebooks. Each reads from `manifests/<run-label>_qc_paths.tsv`.

| Notebook | Theme | Input data |
|---|---|---|
| `1_metrics_comparison.ipynb` | Gene/guide mapping QC metrics | `gene_metrics.tsv`, `guide_metrics.tsv` |
| `2_repression_efficiency_comparison.ipynb` | Intended target knockdown (auROC, auPRC, log2FC) | `intended_target_{results,metrics}.tsv` |
| `3_guide_capture_comparison.ipynb` | Guide representation (cells/guide, UMI, overlap, uniformity) | `per_guide_capture.tsv` |
| `4_deg_comparison.ipynb` | Trans DEG analysis | `trans_{results,metrics}.tsv` |
| `5_guide_level_comparison.ipynb` | Per-guide correlation across technologies | `per_guide_capture.tsv`, `intended_target_results.tsv` |

See `bin/2_qc/README.md` for detailed plot descriptions.

**Supporting scripts in `bin/2_qc/scripts/`:**

- `extract_per_guide_capture.py` — Legacy script to extract per-guide metrics from MuData. Now generated directly by `mapping_guide.py` in the QC pipeline.

## Known Issues

1. **`num_expressed_genes` missing from pipeline outputs** — The GCP pipeline produces `log1p_n_genes_by_counts` but not `num_expressed_genes`. Fixed in local `mapping_gene.py`: auto-derives via `np.expm1()` fallback. The `_local` manifests point to re-run QC with this fix.

2. **Engreitz batch ID mismatch** — Gene and guide modalities use different IGVF analysis set accessions as batch labels (zero overlap). Root cause: CC Perturb-seq library structure uses separate analysis sets for RNA vs guide. Bandaid in local `mapping_guide.py`: copies gene batch labels onto guide obs when mismatch detected. Needs permanent upstream fix in the pipeline sample sheet.

3. **Gersbach_HTv2 GCP naming inconsistency** — Cleanser_unified uses `Gersbach_HTV2` (uppercase V), sceptre_v11 uses `Gersbach_HTv2` (lowercase v). The sync script handles both via alt-name fallback.

4. **Sceptre_v11 vs CLEANSER_unified per-element output differences** — Flag with pipeline team.

   | Feature | cleanser_unified (all 5) | sceptre_v11 (4 of 5) | sceptre_v11 (HTv2 only) |
   |---|---|---|---|
   | NTC elements in per-element results | Yes (5 NTC elements, ~25K–68K rows) | **No** (0 NTC rows) | Yes |
   | `log2_fc_std` column | Yes | **No** | Yes |
   | `intended_target_chr/start/end` columns | Yes | **No** | Yes |
   | Total elements in trans results | 65 | 60 (missing 5 NTCs) | 65 |

   Gersbach HTv2 sceptre_v11 has full output because it was re-run with a newer pipeline version (`Gersbach_HTv2` lowercase v). The other 4 sceptre_v11 datasets used the original pipeline which excluded NTCs and standard errors.

   **Impact:** Empirical null calibration requires z-values (`log2fc / log2fc_std`) from NTC elements. Currently only works for cleanser_unified (5/5 calibrated). Sceptre_v11 is blocked for 4/5 datasets.
